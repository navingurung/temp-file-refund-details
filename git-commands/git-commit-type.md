## 📌 1. Commit Types

| Type      | Description                                      | Example |
|-----------|--------------------------------------------------|---------|
| `feat`    | A new feature                                    | `feat: add user authentication` |
| `fix`     | A bug fix                                        | `fix: resolve login validation error` |
| `docs`    | Documentation changes only                       | `docs: update API guide` |
| `style`   | Code style changes (formatting, no logic change) | `style: format code with Prettier` |
| `refactor`| Code restructuring without behavior change       | `refactor: split data-table component` |
| `perf`    | Performance improvements                         | `perf: optimize table rendering` |
| `test`    | Adding or updating tests                         | `test: add unit tests for export utility` |
| `chore`   | Maintenance tasks (dependencies, configs)        | `chore: update npm dependencies` |
| `build`   | Changes affecting the build system               | `build: update Dockerfile for Node 20` |
| `ci`      | Continuous Integration configuration             | `ci: add GitHub Actions workflow` |
| `revert`  | Revert a previous commit                         | `revert: revert "feat: add export button"` |
